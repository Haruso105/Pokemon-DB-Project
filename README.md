# ポケモン出現データベース(Java Webアプリ)
大学の講義（Webシステム基礎実験）にて制作した、ポケモンの出現情報を管理・分析するためのフルスタックWebアプリケーションです。 Java Servlet/JSPによるサーバーサイドの実装と、リレーショナルデータベース（H2 Database/MySQL）を用いたデータ管理を行っています。

## 主な機能
### 出現情報の管理(CRUD)
- 出現したポケモンの登録、一覧表示、削除機能
### 条件絞り込み・並び替え機能
- ID、番号、名前による昇順・降順ソート。
- タイプの絞り込み：2つのタイプを選択し、特定のタイプを持つ個体のみを検索するフィルタイング機能
### データ登録機能
- 出現シミュレーション：指定した件数分、ランダムなポケモンをランダムな地点に出現・登録させる機能の実装
- 特定のポケモンや場所を指定した状態での一括登録機能

## 仕様技術
**言語**：Java 17(Servlet / JSP)
**データベース**：H2 Database(MySQL)
**サーバー**：Apache Tomcat
**ライブラリ**：JDBC
**データソース**：都道府県・市区町村・ポケモン図鑑データのCSVファイル

## 工夫した点
### 複雑なテーブル結合を用いたデータ抽出
単一のテーブルではなく、APPEAR（出現）、POKEMON（図鑑）、SHI（市区町村）、KEN（都道府県）、TYPE（属性）の5つのテーブルを正規化して設計。NATURAL JOINやINNER JOINを組み合わせた複雑なSQLクエリをJavaから発行し、整合性のあるデータを効率的に取得しています。

### サーバサイドでのフィルタリング機能
SQLだけでは実装が複雑になりがちな「2つのタイプの組み合わせ検索」を、Javaのリスト操作として実装。 取得したオブジェクト群に対して、特定のタイプ文字列が含まれているかを判定する独自メソッドを作成し、柔軟な検索機能を実現しました。

## フォルダ構成
- **src/**：ServletやDAOなどのJavaファイル
- **webapp/**：JSPファイル、およびWEB-INF等のライブラリ
- sql/：データベース構築用SQLスクリプト、及び元データ(csv)
